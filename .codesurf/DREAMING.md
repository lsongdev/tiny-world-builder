## Overview

TinyWorld is a browser-based voxel world builder at `tinyworld.io`. The app is a modular classic-script stack (~40k LoC) served from `dist/` via Netlify. There is no bundler; modules load in numbered order under `engine/world/`. The landing page (`index.html` + `styles/landing.css`) is a separate marketing site. Edits auto-commit and auto-push to production via the CodeSurf/clawd mirror — branches do not guarantee isolation.

## Durable Architecture Facts

### Builder app (`tiny-world-builder.html`)
- Shell HTML with ordered `<script src>` tags; all modules share one global scope
- Module numbering convention: `engine/world/NN-name.js`; naming conflicts throw `SyntaxError` and silently kill the whole module — always use module-local prefixes (e.g., `_fl…` for flight)
- Three.js **r185** vendored at `vendor/three/tinyworld-three.r185.min.js` — do not bump without regenerating via `npm run vendor:three`
- World state: `world[x][z]` (intent) ↔ `cellMeshes['x,z']` (render). Mutate only via `setCell(x, z, opts)` — never write `world[x][z]` directly
- Grid size legal values: `[8, 10, 12, 16, 20]` (cap 20); off-list sizes cause terrain/movement/stargate divergence
- `publish.sh` bakes `dist/`; must run after any view-facing edit before changes appear in the browser
- GitHub Actions CI now exists at `.github/workflows/ci.yml` (added in hardening pass)

### Landing page (`index.html`)
- Hero section uses a static `<img src="assets/landing-hero.png">` — no 3D canvas or WebGL currently in the hero
- The "globe" in user priorities refers to `landing-hero.png` hero image (visually depicts a globe/world)
- Styled by `styles/landing.css`; auth via Netlify Identity

### Engine modules (high watermark: 72)
- `71-infinite-surface.js` — GPU procedural sea during flight; `SEA_Y = -60.3`; `window.__tinyworldInfiniteSurface.{show,hide,tick,isActive}`
- `72-survivor-islands.js` — scatters poser islands across flooded world; pooled + frustum-culled; `window.__tinyworldSurvivorIslands.{show,hide,tick,isActive}`
- `70-animal-anim.js` — grazing animal cosmetic animation; `window.__tinyworldAnimalTick`
- `70-onboarding-driver.js` — driver.js tour; `tinyworld:onboarding:driver.v1`

### Flight sim (`34-flight-sim.js`)

- Pilot names and PvP combat hit relay are now networked — `combat.hit` is gated on active flight state
- `22-audio.js` carries synthesized flight audio: prop drone + wind rush; reactive to throttle/speed; nodes cleaned up on exit
- Propeller axis/pivot derived from mesh bounds (orbital-spin bug fixed)
- Cloud veil reveal smoothed on descent; poser-surface fog now tracks live sky color for day/night consistency
